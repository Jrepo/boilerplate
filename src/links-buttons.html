Buttons and Links

The important thing to remember about links and buttons is that they are two discreet elements with discreet functions.

links link you somewhere.

Buttons however trigger non-linking interactions.

Avoid letting the designs steer you in the direction of mismatching these. Even though a design calls for a link that looks like a chunky button, this style choice shouldn't dictate the html element.

Instead, consider setting up a flexible style system where just adding a class or two can enable a link to look like a button or vice-versa

An example of how this can be done can be found in this repo: src/scss/components/_links-buttons.scss. The key to this approach is making the most of sass's nesting.

*Avoid: Using divs (or other elements) for interactive elements. If you absolutely must, add 'tabindex=0' so that the element gets focus, using this approach will take more testing and likely some custom js to ensure the element is fully keyboard operable.
